## 3. Відомі обмеження (що інакше, ніж на Webflow)

- **Карта `/some-map`** — це Leaflet (custom embed), не Webflow-функція. Працює й далі, але залежить від **зовнішніх** `unpkg.com` (Leaflet) і `gyborg.github.io` (тайли — окремий картографічний проєкт). Не локалізовано свідомо (та ж логіка, що й для `archive.*`). Якщо ці сервіси стануть недоступні — карта не покаже тайли; сторінки об'єктів `map/…` лишаться робочими.
- **`archive.memory.org.ua`** — окремий проєкт, не чіпали; посилання абсолютні.
- **Форма «Додати історію»** тепер відкриває поштовий клієнт (`mailto`), а не вебформу. За бажання згодом — Google Form / Formspree.
- **Дрібні пре-існуючі заглушки** з Webflow («Збір коштів на ЗСУ» у футері, дисклеймер-текст у `<a href="#">`) лишені як в оригіналі. Деталі: [`qa-report.md`](qa-report.md).
- Редактор/CMS Webflow більше недоступний — правки контенту тепер через HTML у репозиторії.

## 4. Деплой (потрібна ваша дія)

Сайт зібрано в `~/memory-org-ua/site`. Для завершення потрібна авторизація в GitHub — її я за вас виконати не можу.

**Крок 1 — авторизуйтесь у своєму терміналі:**
```bash
gh auth login
```
**Крок 2 — повідомте мені ваш GitHub-username** (і публічний чи приватний репозиторій) — і я створю репозиторій `memory-org-ua`, запушу й увімкну Pages. Або зробіть самі:
```bash
cd ~/memory-org-ua/site && gh repo create memory-org-ua --public --source=. --remote=origin --push
```
**Крок 3 — увімкніть Pages:** Settings → Pages → Source **Deploy from a branch** → Branch **main**, папка **/(root)** → Save.
> GitHub Pages віддає сайт лише з `/(root)` або `/docs` — теки `/site` немає. Тому вміст сайту лежить у **корені** репозиторію, а звіти — в `docs/`.

Далі сайт відкриється на `https://artur-blip.github.io/memory-org-ua/` — перевірте роботу (ще до зміни DNS).

## 5. Інструкція зі зміни DNS (робите ви, у реєстратора)

Коли перевірили `*.github.io` — переходьте на кастомний домен. У DNS `memory.org.ua`:

**A) Apex `memory.org.ua` → 4 A-записи GitHub Pages:**
```
A   @   185.199.108.153
A   @   185.199.109.153
A   @   185.199.110.153
A   @   185.199.111.153
```
(Опційно AAAA/IPv6: `2606:50c0:8000::153`, `…8001::153`, `…8002::153`, `…8003::153`.)

**B) `www` → GitHub Pages:**
```
CNAME   www   artur-blip.github.io.
```
(`artur-blip` — ваш GitHub-логін.)

> Файл `CNAME` містить `www.memory.org.ua` — тобто канонічний домен — **www**, apex редіректить на www. Хочете навпаки (apex головний) — скажіть, поміняю `CNAME` на `memory.org.ua`.

**C) Enforce HTTPS:** у Settings → Pages, коли домен підтвердиться і випуститься сертифікат (до ~24 год), поставте галочку **Enforce HTTPS**.

**Обережно:** приберіть старі A/CNAME на Webflow для `@` і `www`. **Не чіпайте** запис `archive` (окремий проєкт).

## 6. Порядок дій і застереження

1. Задеплоїти й перевірити на `*.github.io`.
2. Змінити DNS (розділ 5). Поширення — від хвилин до 24–48 год.
3. Дочекатися HTTPS-сертифіката й увімкнути **Enforce HTTPS**.
4. **Не скасовуйте підписку Webflow одразу.** Лише **після** зміни DNS і **щонайменше тижня** стабільної роботи на GitHub Pages.
